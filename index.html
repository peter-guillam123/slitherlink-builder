Draw your loop by clicking the grid edges.

Click an edge to add it to your loop — or click-and-drag to paint a line; drag back over it to erase. The loop can’t fork — each dot joins at most two lines. Build one single closed loop, then press Create unique puzzle: it fills every clue, then quietly removes as many as it can while the puzzle still has exactly one solution. After that, click any number to hide or restore it — the difficulty re-reads as you go. Numbers track the loop live if you redraw it. ⌘Z undoes, ⇧⌘Z redoes.

About this builder

A small workshop tool for making Slitherlink puzzles by hand: draw the loop you want, and it works out a clean set of numbers that has your loop as its one and only answer.

How it's made

It's a single HTML file — no install, no account, nothing leaves your machine. The grid is SVG; the hard part, checking that a set of clues has exactly one solution, runs in a background worker so the interface never freezes. That solver does proper Slitherlink deduction (forced edges, vertex parity, no premature little loops) rather than blind trial and error, which is why a 6×6 now takes a few milliseconds instead of the best part of an hour.

The diary

23 June 2026

Tuned for thumbs

Drawing on a phone was fiddlier than it should have been — the edge targets were tuned for a mouse pointer, not a fingertip, and the board could sit slightly too wide and need awkward sideways scrolling. Now the board scales to fit the screen, the grab area around each edge is wider on touch (and measured in screen pixels, so it feels the same however large the board renders), and a small wobble between finger-down and finger-up no longer paints the edge next door. Dragging a line with one finger works properly.

23 June 2026

Alan’s notes, round one

A page of pencilled notes from the puzzles desk, mostly small and all sensible. The lattice dots are black now, and the clue numbers have swapped the decorative serif for a plainer, more legible face — they’re puzzle furniture, not a headline. Saved files and exported images carry the date in their name, so a library doesn’t collapse into a pile of identical filenames.

Two changes go a bit deeper. The loop can no longer fork: a dot will only ever join two lines, so you can’t accidentally draw something that isn’t a valid path. And once numbers are on the board they track the loop live as you redraw it — change the shape and the clues follow — while the difficulty re-reads itself every time you hide or restore a number by hand.

23 June 2026 Milestone

A rebuild, and the end of the hour-long wait

The first version worked but was painfully slow — a 6×6 could take an hour in Safari, and anything larger was hopeless. The culprit wasn't the browser; it was the solver, which brute-forced every edge with almost no deduction and re-scanned the whole grid millions of times over.

This rebuild replaces it with a real propagation solver: it deduces the forced moves a human would, uses a union-find to reject loops that close too early, and runs off the main thread. Before trusting it, it was checked against an exhaustive brute force on thousands of small grids — and it caught a genuine bug, where a loop with a stray tail was being counted as valid. Same buttons as before, but a 6×6 is now instant and 10×10 is comfortable.

23 June 2026

The niceties

Added the things that make it pleasant rather than merely fast: undo and redo while drawing, a rough difficulty read on the finished puzzle, save and load so you can keep a library, and a clean print layout for the page or a PDF. Drawing now supports click-and-drag, so laying down a long edge is one gesture, not twenty clicks.
